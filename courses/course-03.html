Django 的 MTV 设计模式

2.1 经典的 MVC 设计模式

MVC 即 Model-View-Controller（模型-视图-控制器），是经典的软件开发设计模式。

组件说明
Model（模型）数据模型。模型不是数据本身，而是抽象的描述数据的构成和逻辑关系。通常模型包括了数据表的各个字段和相互关系。Web 开发框架会根据模型的定义来自动生成数据表。
View（视图）主要用于显示数据，用来展示用户可以看到的内容或提供用户可以输入或操作的界面。数据来源于数据库，用户输入的数据给控制器。
Controller（控制器）应用程序中处理用户交互的部分。通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据。

MVC 的比喻：如果把 MVC 比喻成一个粽子，那么 View 就是最外面一层的绿色玉米叶，是吃货们可以直接看到的。Controller 就是中间那层熟糯米，而粽子的核心自然是最里面那一层的肉馅 Model 模型了。

2.2 Django 的 MVT 设计模式

Django 的 MVT 设计模式由 Model（模型）、View（视图）和 Template（模板）三部分组成，分别对应单个 app 目录下的 models.py、views.py 和 templates 文件夹。

Django MVT经典 MVC说明
ModelModel这个与经典 MVC 模式下的模型 Model 差不多。
ViewController这个与 MVC 下的控制器 Controller 更像。视图不仅负责根据用户请求从数据库读取数据、指定向用户展示数据的方式，还可以指定渲染模板并处理用户提交的数据。
TemplateView这个与经典 MVC 模式下的视图 View 一致。模板用来呈现 Django view 传来的数据，也决定了用户界面的外观。

2.3 Django MVT 工作流程

Django MVT 设计模式中最重要的是视图（view），因为它同时与模型（model）和模板（templates）进行交互。当用户发来一个请求（request）时，Django 会对请求头信息进行解析，解析出用户需要访问的 URL 地址，然后根据路由 urls.py 中的定义的对应关系把请求转发到相应的视图处理。视图会从数据库读取需要的数据，指定渲染模板，最后返回响应数据。

💡 工作流程：用户请求 → URL 路由 → 视图处理 → 数据库查询 → 模板渲染 → 返回响应

Django 的路由系统（URLconf）

Django 的项目文件夹和每个应用（app）目录下都可以有 urls.py 文件，它们共同构成了 Django 的路由配置系统（URLconf）。服务器收到用户请求后，会根据用户请求的 URL 地址和 urls.py 中配置的 URL-视图映射关系，去调用执行相应的视图函数或视图类，最后由视图返回给客户端数据。

一个优美的 URL 不仅层次分明、逻辑清晰，而且便于记忆，也更利于搜索引擎收录。相反，一个糟糕的 URL 往往可读性差、维护成本高，还更容易造成程序冲突。

3.1 URLconf 是如何工作的？

假如我们有一个 blog 博客应用，需要编写两个视图函数，一个用于展示文章列表，一个用于展示文章详情，那么对应的 urls.py 和 views.py 正常情况下可以写成下面这样：

Python
# blog/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('blog/', views.index),
    path('blog/articles/<int:id>/', views.article_detail),
]

# blog/views.py
def index(request):
    # 展示所有文章
    pass

def article_detail(request, id):
    # 展示某篇具体文章
    pass

那么上面这段代码是如何工作的呢？

  • 当用户在浏览器中输入 /blog/ 时，URLconf 会调用 views.py 里的 index 方法，用于展示所有文章。
  • 当用户在浏览器中输入 /blog/articles/<int:id>/ 时，URLconf 不仅会调用 views.py 里的 article_detail 方法，而且还会把文章 id 通过尖括号的形式传递给视图。
  • 在这套机制里，urlpatterns 列表起到了决定性作用，它本质上就是一个 URL-视图映射关系表，负责请求调度。
💡 注意：当你配置应用级 URL 时，别忘了把应用自己的 urls.py 再加入到项目级的 URL 配置中。
Python
# mysite/urls.py
from django.urls import include, path

urlpatterns = [
    path('', include('blog.urls')),
    # ...
]

3.2 path() 和 re_path() 方法

写一个 URL 并不难，难点在于如何把参数从 URL 传递给视图。Django 提供了两种常见的路由设计方法：path() 和 re_path()，它们都支持向视图函数或类传递参数。

  • path()：采用 <变量类型:变量名> 或 <变量名> 的方式传递参数，例如 <int:id>、<slug:slug> 或 <username>。
  • re_path()：采用命名组 (?P<变量名>表达式) 的方式传递参数，适合更复杂的正则匹配场景。

下例中，我们分别用 path() 和 re_path() 定义了两个等效的 URL，它们都把文章的整数型 id 传递给了同一个视图：

Python
# blog/urls.py
from django.urls import path, re_path
from . import views

# 写法一：path()
urlpatterns = [
    path('blog/articles/<int:id>/', views.article_detail, name='article_detail'),
]

# 写法二：re_path()
urlpatterns = [
    re_path(r'^blog/articles/(?P<id>\d+)/$', views.article_detail, name='article_detail'),
]

# blog/views.py
def article_detail(request, id):
    # 展示某篇文章
    pass
  • URL 中的参数要用尖括号，而不是圆括号。
  • 匹配模式最开头通常不需要加斜杠 /，但建议以斜杠结尾，这样风格更统一。
  • 使用 re_path() 时并不一定总要以 $ 结尾；如果它只是一个前缀入口，就不能过早截断。
Python
from django.urls import include, re_path

urlpatterns = [
    re_path(r'^blog/', include('blog.urls')),
    # ...
]

Django 的视图（View）

4.1 什么是视图及其工作原理

Django 的 Web 开发也遵循经典软件设计 MVC 模式开发的。View（视图）主要根据用户的请求返回数据，用来展示用户可以看到的内容（比如网页、图片），也可以用来处理用户提交的数据，比如保存到数据库中。

Django 的视图（views.py）通常和 URL 路由（URLconf）一起工作的。服务器在收到用户通过浏览器发来的请求后，会根据用户请求的 URL 地址和 urls.py 里配置的 URL-视图映射关系，去执行相应视图函数或视图类，从而返回给客户端响应数据。

4.2 最简单的函数视图

当用户发来一个请求 request 时，我们通过 HttpResponse 打印出 "Hello, World!"。

Python
# views.py
from django.http import HttpResponse

def index(request):
    return HttpResponse("Hello, World!")
💡 重要提示：每个视图函数的第一个默认参数都必需是 request，它是一个全局变量。Django 把每个用户请求封装成了 request 对象，它包含了当前请求的所有信息，比如请求路径 request.path、当前用户 request.user 以及用户通过 POST 提交的数据 request.POST。

Django 模型

Model（模型）简而言之即数据模型，是一个 Django 应用的核心。模型不是数据本身，而是抽象的描述数据的构成和逻辑关系。

每个 Django 的模型（model）实际上是个类，继承了 models.Model。每个 Model 应该包括属性（字段）、关系和方法。当你定义好 Model 模型后，Django 的接口会自动帮你在数据库生成相应的数据表。

5.1 模型的组成

一个标准的 Django 模型分别由模型字段、META 选项和方法三部分组成。Django 官方编码规范建议按如下方式排列：

  1. 定义的模型字段：包括基础字段和关系字段
  2. 自定义的 Manager 方法
  3. class Meta 选项（可选）
  4. def __str__()（可选）
  5. def save()（可选）
  6. def get_absolute_url()（可选）
  7. 其它自定义的方法

5.2 模型的基础字段

字段类型说明
CharField()需要通过 max_length 设置最大字符长度。可设置 blank=True、default=''、unique=True
TextField()适合大量文本，max_length 选项可选
DateField() 和 DateTimeField()可设置 default=timezone.now、auto_now=True、auto_now_add=True
EmailField()用于邮箱字段，建议设置 unique=True
IntegerField(), BooleanField()可设置 blank=True、null=True、default
FileField(), ImageField()需要设置 upload_to 指定上传路径

5.3 模型的关系字段

关系类型说明
OneToOneField()单对单关系 - 必需指定 on_delete 选项，可设置 related_name
ForeignKey()单对多关系 - 必需指定 on_delete 选项，可设置 related_name
ManyToManyField()多对多关系 - 可设置 through 建立中间模型，可设置 related_name

5.4 多对多关系示例

Python
from django.db import models

class Person(models.Model):
    name = models.CharField(max_length=128)
    def __str__(self):
        return self.name

class Group(models.Model):
    name = models.CharField(max_length=128)
    members = models.ManyToManyField(Person, through='Membership')

class Membership(models.Model):
    person = models.ForeignKey(Person, on_delete=models.CASCADE)
    group = models.ForeignKey(Group, on_delete=models.CASCADE)
    date_joined = models.DateField()
    invite_reason = models.CharField(max_length=64)

5.5 on_delete 删除选项

选项说明
CASCADE级联删除。删除关联记录时，所有相关记录也会被删除
PROTECT保护模式。如果有外键关联，就不允许删除
SET_NULL置空模式。删除时，外键字段会被设置为空
SET_DEFAULT置默认值。删除时，外键字段设置为默认值
DO_NOTHING什么也不做。删除不报错，外键值依然保留

5.6 related_name 选项

related_name 用于设置模型反向查询的名字。Django 默认使用 模型名_set 进行反查，但我们可以通过 related_name 设置更友好的名字。

Python
class Publisher(models.Model):
    name = models.CharField(max_length=30)

class Book(models.Model):
    name = models.CharField(max_length=30)
    publisher = models.ForeignKey(Publisher, on_delete=models.CASCADE, related_name='books')

# 反向查询：publisher.books.all()  而不是 publisher.book_set.all()

5.7 模型的 META 选项

选项说明
abstract=True指定该模型为抽象模型
verbose_name为模型设置便于人类阅读的别名
db_table自定义数据表名
ordering自定义按哪个字段排序，- 代表逆序
indexes为数据表设置索引
constraints给数据库中的数据表增加约束

5.8 模型的标准方法

  • def __str__()：给单个模型对象实例设置人为可读的名字
  • def save()：重写 save 方法
  • def get_absolute_url()：为单个模型实例对象生成独一无二的 URL

def create_post(request):
    """创建新文章"""
    if request.method == 'POST':
        title = request.POST.get('title')
        content = request.POST.get('content')
        author = request.POST.get('author', '匿名')

def delete_by_author(request):
    """删除指定作者的所有文章"""
    if request.method == 'POST':
        author = request.POST.get('author')
        if author:
            # 使用 filter().delete() 删除多条数据
            deleted_count, _ = Post.objects.filter(author=author).delete()
            return redirect('post_list')
    
    # 获取所有作者列表
    authors = Post.objects.values_list('author', flat=True).distinct()
    return render(request, 'blog/delete_by_author.html', {'authors': authors})

def edit_post(request, post_id):
    """编辑并更新文章"""
    post = get_object_or_404(Post, id=post_id)

def update_author(request):
    """批量更新文章作者"""
    if request.method == 'POST':
        old_author = request.POST.get('old_author')
        new_author = request.POST.get('new_author')
        
        if old_author and new_author:
            # 使用 filter().update() 批量更新

            return redirect('post_list')
    
    # 获取所有作者列表
    authors = Post.objects.values_list('author', flat=True).distinct()
    return render(request, 'blog/update_author.html', {'authors': authors})

Count 方法（聚合统计）

from django.db.models import Count

def statistics(request):
    """显示博客统计信息"""
    # 总文章数
    total_posts = Post.objects.count()
    
    # 总作者数
    total_authors = Post.objects.values('author').distinct().count()
    
    # 每个作者的文章数（使用 Count 聚合）
    author_stats = Post.objects.values('author').annotate(count=Count('id')).order_by('-count')
    
    # 最新的 5 篇文章
    latest_posts = Post.objects.all()[:5]
    
    context = {
        'total_posts': total_posts,
        'total_authors': total_authors,
        'author_stats': author_stats,
        'latest_posts': latest_posts,
    }
    
    return render(request, 'blog/statistics.html', context)

  • 入门使用 create() 方法创建 3 篇文章，分别由不同作者撰写。提示：Post.objects.create(title=..., content=..., author=...)
  • 入门查询所有文章，并按创建时间逆序排列，显示最新的 5 篇。提示：Post.objects.all()[:5]
  • 进阶使用 filter() 查询标题包含"Django"的所有文章，并统计数量。提示：filter(title__icontains='Django').count()
  • 进阶使用 filter().update() 批量更新某个作者的所有文章作者名为"新作者"。提示：filter(author='old_author').update(author='new_author')
  • 挑战使用 Q 方法查询标题包含"Python"或"Django"，且作者为"张三"的文章。提示：filter(Q(title__icontains='Python') | Q(title__icontains='Django'), author='张三')
