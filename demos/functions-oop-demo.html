函数与面向对象 - 交互演示

本页面专为有 C 语言基础的同学设计，通过对比和可视化帮助你理解 Python 函数和面向对象的特性。

1. 函数：从 C 到 Python

🎯 核心区别 C 语言的函数像「严格的工厂流水线」：必须声明输入输出类型，只能返回一个值。
Python 的函数像「灵活的工作台」：不需要声明类型，可以返回多个值，函数本身也是对象。
特性 C 语言 Python
返回类型 必须声明 int func() 不需要声明
参数类型 必须声明 不需要（可选类型注解）
默认参数 ❌ 不支持 ✅ 支持
可变参数 va_list 复杂 *args, **kwargs 简洁
返回多值 只能返回一个（靠指针模拟） ✅ 返回元组
函数作为参数 函数指针 ✅ 一等公民

图解：Python 如何返回多个值？

💡 类比 C 语言返回多值就像「一个人只能拿一个包裹」，需要用指针传递额外的"包裹"。
Python 返回多值就像「把多个物品打包成一个快递箱」，一次性返回一个元组。
🔄 多返回值步进器
步骤 1/5
🎯 关键点：Python 的多返回值本质是返回一个元组，然后通过「解包」赋值给多个变量。这比 C 语言用指针传递输出参数更直观。

图解：作用域 LEGB 规则

💡 类比：找人的顺序 Python 找一个变量，就像你找一个人：先找自己房间 (Local) → 再找隔壁房间 (Enclosing) → 再找整栋楼 (Global) → 最后找全国 (Built-in)。找到了就用，找不到就报错 NameError。
🔍LEGB 作用域步进器
步骤 1/6
🎯 关键点：C 语言也有类似的作用域规则（块作用域 → 函数作用域 → 全局）。Python 的 LEGB 多了「闭包」这一层（E），这是 C 语言没有的。

图解：*args 和 **kwargs

💡 类比：打包行李 *args 像「随便塞进行李箱的东西」—— 按顺序打包成一个列表（元组）。
**kwargs 像「贴了标签的物品」—— 用名字索引，打包成字典。
📦参数打包/解包可视化

2. 面向对象：从 C 结构体到 Python 类

💡 你已经知道的：C 语言的结构体 C 的 struct 可以把相关数据打包在一起，但操作这些数据的函数是独立的。
Python 的 class 把数据（属性）和操作它的函数（方法）打包在一起，更内聚。

C 语言：数据与函数分离

typedef struct {
    char name[50];
    int age;
    float score;
} Student;

// 函数在结构体外面
void introduce(Student *s) {
    printf("我是%s，%d岁\n",
           s->name, s->age);
}

int is_passed(Student *s) {
    return s->score >= 60;
}

// 使用
Student s = {"Alice", 20, 90};
introduce(&s);

Python：数据与方法在一起

class Student:
    def __init__(self, name,
                 age, score=0):
        self.name = name
        self.age = age
        self.score = score

    # 方法在类内部
    def introduce(self):
        print(f"我是{self.name}，"
              f"{self.age}岁")

    def is_passed(self):
        return self.score >= 60

# 使用
s = Student("Alice", 20, 90)
s.introduce()

步骤 1/5
🎯 对比：self 就是 C 语言里的 this 指针，只是 Python 不隐藏它，必须显式写出来。__init__ 就是 C++ 的构造函数。

2.1 封装：保护内部数据

💡 类比：ATM 机 你不能直接修改银行数据库里的余额（相当于 _balance 私有），只能通过存款/取款接口操作（相当于 deposit() / withdraw()）。
🏦BankAccount 封装演示

2.2 继承：代码复用

💡 类比：公司职级 Employee（员工）是基类，有 name / salary / work()。
Manager（经理）继承 Employee，额外有 manage() 方法。
Engineer（工程师）继承 Employee，额外有 code() 方法。
🌳继承关系可视化
🎯 关键点：子类自动拥有父类的所有属性和方法。super().__init__() 相当于「先初始化父类部分」，就像 C++ 的成员初始化列表调用基类构造函数。

2.3 多态：同一接口，不同行为

💡 类比：遥控器 同样按「播放」按钮，接的是电视就播电视，接的是音响就播音乐。
多态让你写 animal.speak()，不管 animal 是 Dog 还是 Cat，都会调用各自的叫声。

完整的 OOP 示例：学生管理

综合运用封装、继承、多态
class Person:
    def __init__(self, name, age):
        self.name = name
        self._age = age  # 封装

    @property
    def age(self):
        return self._age

    def __str__(self):
        return f"{self.name}({self._age}岁)"

class Student(Person):  # 继承
    def __init__(self, name, age, student_id):
        super().__init__(name, age)  # 调用父类构造
        self.student_id = student_id
        self._scores = []

    def add_score(self, subject, score):
        self._scores.append((subject, score))

    @property
    def gpa(self):
        if not self._scores:
            return 0
        return sum(s for _, s in self._scores) / len(self._scores)

    def report(self):  # 多态：子类有自己的实现
        return f"学生 {self} - 平均分: {self.gpa:.1f}"

class GradStudent(Student):
    def __init__(self, name, age, student_id, advisor):
        super().__init__(name, age, student_id)
        self.advisor = advisor

    def report(self):  # 覆盖父类方法
        base = super().report()
        return f"{base} - 导师: {self.advisor}"

# 多态使用
people = [
    Student("Alice", 20, "S001"),
    GradStudent("Bob", 24, "G001", "Dr. Wang")
]
people[0].add_score("数学", 90)
people[0].add_score("英语", 85)

for p in people:  # 同一接口，不同行为
    print(p.report())
