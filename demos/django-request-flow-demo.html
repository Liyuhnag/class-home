Django 请求全流程演示

点击「下一步」逐步追踪一条 HTTP 请求从浏览器到 Django 服务器再返回响应的完整链路

🌐
客户端用户在浏览器发起 HTTP 请求
用户在浏览器地址栏输入 URL，或点击页面上的链接，浏览器随即发起一个 HTTP GET/POST 请求，并附带请求头（Headers）、Cookie、请求体（Body）等信息。
# 浏览器发出的 HTTP 请求报文
GET /blog/post/5/ HTTP/1.1
Host: www.example.com
Accept: text/html
Cookie: sessionid=abc123
User-Agent: Mozilla/5.0
HTTP Request → www.example.com/blog/post/5/
⚙️
Django 服务器WSGI / ASGI 接收并封装请求
Django 通过 WSGI（同步）或 ASGI（异步）服务器接收原始 HTTP 请求，将其封装成 HttpRequest 对象，该对象包含了请求的所有信息。
# Django 将请求封装为 HttpRequest 对象
request.method    # 'GET' 或 'POST'
request.path      # '/blog/post/5/'
request.GET       # 查询参数 QueryDict
request.POST      # POST 数据 QueryDict
request.user      # 当前登录用户
request.COOKIES   # Cookie 字典
request.session   # Session 数据
HttpRequest 对象已创建
🔒
中间件Middleware 请求阶段处理
请求依次经过 settings.py 中配置的中间件列表（从上到下）。每个中间件可以对请求进行预处理，例如身份认证、Session 解析、CSRF 验证等。
# settings.py 中的中间件配置
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
]
