Python 装饰器 — 交互演示

装饰器 = 接收函数作为参数、返回新函数的高阶函数。Django 大量使用：@login_required、@api_view、@cache_page。
💡 前置知识：装饰器由闭包实现，建议先看 函数演示页第 7 节「闭包 → 装饰器」。

2. 带参数的装饰器

💡 三层嵌套 普通装饰器两层：decorator → wrapper。
带参数装饰器三层：outer(参数) → decorator(func) → wrapper(*args)。
📋带参数装饰器代码
from functools import wraps

def retry(max_retries=3, exceptions=(Exception,)):
    """带参数装饰器：失败时自动重试"""
    def decorator(func):          # 第二层：接收函数
        @wraps(func)
        def wrapper(*args, **kwargs):  # 第三层：执行逻辑
            for attempt in range(1, max_retries + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    print(f"第 {attempt} 次失败: {e}")
                    if attempt == max_retries:
                        raise
            return wrapper
        return decorator          # 返回真正的装饰器

@retry(max_retries=3, exceptions=(ConnectionError,))
def fetch_data(url):
    ...  # 模拟不稳定的网络请求

3. 多个装饰器叠加

🥪叠加顺序可视化

装饰器从下往上应用，调用时从外往里执行。

代码

@bold          ← 最后应用
@italic        ← 先应用
def greet(name):
    return f"Hello, {name}"

等价于：bold(italic(greet))

调用栈（点击演示）

bold.wrapper("Alice")
  italic.wrapper("Alice")
    greet("Alice") → "Hello, Alice"
  italic 返回 "<i>Hello, Alice</i>"
bold 返回 "<b><i>Hello, Alice</i></b>"

4. 常用内置装饰器 & @lru_cache

装饰器来源用途
@property内置属性访问器，保护私有数据
@classmethod内置类方法，第一个参数是 cls
@staticmethod内置静态方法，不依赖实例或类
@functools.wrapsfunctools保留被装饰函数的元信息
@functools.lru_cachefunctools自动缓存函数结果（备忘录）
