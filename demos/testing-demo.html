单元测试与 pytest

写完代码怎么知道它是对的？用眼睛看？手动运行一遍？这些方法在项目变大后就不够了。自动化测试是专业开发的必备技能。

为什么要写测试？

🏗️ 类比：盖房子的质检
盖房子时，每砌一层砖都要检查是否水平。如果等整栋楼盖完才发现第 3 层歪了，拆掉重来的代价巨大。
写测试就是写代码时随时质检：每写一个功能，立刻验证它是否正确。改代码后一键运行所有测试，几秒内知道有没有搞坏别的功能。

🐛 没有测试会怎样？

pytest 入门：写你的第一个测试

🆚 unittest vs pytest
Python 内置了 unittest（需要写类、用 self.assertEqual()），但社区更推荐 pytest：
✅ 直接写函数，不用写类
✅ 用 Python 原生 assert，不用记一堆 self.assertXxx
✅ 报错信息超级详细，一眼看出哪里不对
✅ 丰富的插件生态（Django、异步、覆盖率...）

▶️ 运行你的第一个 pytest

# test_math.py

def add(a, b):
    return a + b

def divide(a, b):
    if b == 0:
        raise ValueError("除数不能为0")
    return a / b

# ---------- 测试 ----------

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

def test_divide():
    assert divide(10, 2) == 5.0

def test_divide_by_zero():
    import pytest
    with pytest.raises(ValueError):
        divide(10, 0)
点击上方按钮运行测试...
pytest 的约定：
1. 测试文件命名 test_*.py 或 *_test.py
2. 测试函数命名 test_*
3. 用 assert 断言，不需要 import 任何东西
4. 运行：pytest（自动发现所有测试）

pytest 断言速查

pytest 最大的优点：直接用 assert，不需要记 self.assertEqual 之类的。点击查看每种断言的用法：

assert a == b
相等判断
assert a != b
不相等判断
assert x
真值判断
assert a in b
包含判断
assert isinstance(x, T)
类型判断
pytest.raises(Exc)
异常判断
assert a == pytest.approx(b)
浮点近似
assert len(x) == n
长度判断

进阶：参数化测试

📋 一个函数，多组数据
如果要测试 add(1,2)==3、add(-1,-2)==-3、add(0,0)==0...
不需要写三个测试函数！用 @pytest.mark.parametrize 一次搞定。

📊 参数化测试演示

import pytest

def is_even(n):
    return n % 2 == 0

@pytest.mark.parametrize("number, expected", [
    (2, True),     # 正偶数
    (3, False),    # 正奇数
    (0, True),     # 零
    (-4, True),    # 负偶数
    (-7, False),   # 负奇数
])
def test_is_even(number, expected):
    assert is_even(number) == expected

进阶：Fixture（测试夹具）

🧹 类比：做实验前要准备器材
做化学实验前要准备烧杯、试剂（setup）；实验结束后要洗烧杯、清理桌面（teardown）。
pytest 的 fixture 就是自动帮你做「准备」和「清理」的工具。

🔧 Fixture 执行流程

import pytest

@pytest.fixture
def db_connection():
    conn = create_connection()  # ← setup: 创建连接
    yield conn                 # ← 把连接交给测试用
    conn.close()               # ← teardown: 关闭连接

def test_query(db_connection):  # ← 自动注入 fixture
    result = db_connection.execute("SELECT 1")
    assert result == 1
📋 发现 fixture pytest 发现 test_query 需要 db_connection fixture 自动注入
🔧 Setup 执行 fixture 函数到 yield 之前：创建数据库连接 create_connection()
🧪 运行测试 yield 的值作为参数传给 test_query，执行测试代码 assert result == 1
🧹 Teardown 测试结束后，执行 yield 之后的代码：关闭连接 conn.close()
✅ 完成 测试通过！fixture 保证了每次测试都有干净的环境 PASSED

数据库连接 / 测试用户 / 临时文件 / API 客户端 / Mock 对象

完整实战：测试一个用户注册模块

🚀 实战演示

# user.py — 业务代码
import re

def validate_email(email):
    if not re.match(r'^[\w.+-]+@[\w-]+\.[\w.]+$', email):
        raise ValueError("邮箱格式无效")
    return True

def validate_password(password):
    if len(password) < 8:
        raise ValueError("密码至少8位")
    if not re.search(r'\d', password):
        raise ValueError("密码必须包含数字")
    return True

# test_user.py — 测试代码
import pytest

class TestValidateEmail:
    def test_valid_email(self):
        assert validate_email("alice@example.com") == True

    def test_invalid_email(self):
        with pytest.raises(ValueError, match="邮箱格式无效"):
            validate_email("not-an-email")

@pytest.mark.parametrize("password, error_msg", [
    ("short", "密码至少8位"),
    ("nonnumber", "密码必须包含数字"),
])
def test_invalid_password(password, error_msg):
    with pytest.raises(ValueError, match=error_msg):
        validate_password(password)

def test_valid_password():
    assert validate_password("secure123") == True

pytest 常用命令速查

命令作用
pytest运行当前目录下所有测试
pytest test_math.py运行指定文件
pytest -v详细输出（显示每个测试的名字和结果）
pytest -x遇到第一个失败就停止
pytest -k "email"只运行名字包含 "email" 的测试
pytest --tb=short简短的错误堆栈
pytest --cov=myapp测试覆盖率报告（需安装 pytest-cov）
pip install pytest pytest-django安装 pytest 和 Django 插件
Django 项目中使用 pytest：
1. pip install pytest-django
2. 创建 pytest.ini，写入 [pytest] 和 DJANGO_SETTINGS_MODULE = myproject.settings
3. 运行 pytest（替代 python manage.py test）
